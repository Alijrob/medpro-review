# Session Summary - 2026-05-27

## Title
Phase 6-A complete: data stores live on Hostinger (Postgres 16 + Redis 7, all 11 migrations applied)

## Summary
Phase 6-A deployed the medpro-review data layer to Hostinger (147.93.119.147). The original plan (DECISIONS.md Entry 042) called for Docker Compose for Postgres and Redis; on inspection, Hostinger was already running native Postgres 16 with 13 application databases and no Docker installed, so the approach was adapted: the existing native Postgres received new medpro and medpro_audit databases under a new medpro_admin role, and Redis 7 was installed via apt. This deviation is documented in DECISIONS.md Entry 043. Running the Alembic migrations for the first time against a real Postgres instance exposed six categories of bugs that had been latent in the migration files: (1) NOW() AT TIME ZONE 'UTC' in DEFAULT clauses requires parentheses to be parsed correctly, (2) plain Python string server_defaults with embedded SQL quotes need sa.text() wrapping or SQLAlchemy double-escapes them, (3) op.create_index() does not accept a comment= keyword argument, (4) the migration runner needed CREATEROLE privilege, (5) CREATE ROLE IF NOT EXISTS is not valid PostgreSQL syntax and was replaced with a DO block and pg_roles check, and (6) ALTER ROLE NOSUPERUSER requires SUPERUSER privilege. All six categories were fixed, committed, and pushed across nine commits. After all fixes, alembic upgrade head ran cleanly through all 11 migrations, producing 13 tables and 33 source health records in the medpro DB. Redis is running with password auth, 256MB cap, and allkeys-lru eviction. Both services are bound to 127.0.0.1 only. Poetry was installed on Hostinger and the project's production dependencies were installed. All 105 data/migration tests still pass locally.
